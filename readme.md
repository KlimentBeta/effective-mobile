# Effective Mobile — Система аутентификации и авторизации

## 📋 Описание

Backend-приложение с собственной системой аутентификации и авторизации. Реализовано разграничение доступа к ресурсам на основе ролей пользователей.

---

## 🗄️ Схема базы данных

### 1. Таблица `users` — Пользователи

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INT (PK) | Уникальный идентификатор |
| `last_name` | VARCHAR(50) | Фамилия |
| `first_name` | VARCHAR(50) | Имя |
| `middle_name` | VARCHAR(50) | Отчество (опционально) |
| `email` | VARCHAR(255) | Email (уникальный, для входа) |
| `password_hash` | VARCHAR(255) | Хешированный пароль |
| `is_active` | BOOLEAN | Статус аккаунта (True — активен) |
| `created_at` | DATETIME | Дата создания |
| `updated_at` | DATETIME | Дата обновления |

---

### 2. Таблица `roles` — Роли

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INT (PK) | Уникальный идентификатор |
| `name` | VARCHAR(50) | Название роли (уникальное) |

**Примеры ролей:**
- `admin` — полный доступ
- `manager` — расширенный доступ
- `user` — базовый доступ

---

### 3. Таблица `user_roles` — Связь пользователь ↔ роль

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INT (PK) | Уникальный идентификатор |
| `user_id` | INT (FK → users) | Ссылка на пользователя |
| `role_id` | INT (FK → roles) | Ссылка на роль |

**Ограничение:** Уникальная пара `(user_id, role_id)`

---

### 4. Таблица `posts` — Бизнес-элементы (Посты)

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INT (PK) | Уникальный идентификатор |
| `title` | VARCHAR(100) | Заголовок поста |
| `description` | TEXT(500) | Описание поста |
| `created_at` | DATETIME | Дата создания |

---

## 🚀 Запуск проекта

### 1. Подготовка окружения


#### Перейдите в папку проекта
```
cd effective-mobile
```

#### Создайте виртуальное окружение
```
python -m venv venv
```

#### Активируйте виртуальное окружение
#### Linux / macOS:
```
source venv/bin/activate
```
#### Windows:
```
venv\Scripts\activate
```

#### Установите зависимости
```
pip install django djangorestframework PyJWT psycopg2-binary
```

#### Настройка базы данных
Подключить в файле settings.py бд PostgreSQL

#### Создать миграции для приложения core
```
python manage.py makemigrations core
```
#### Применить все миграции
```
python manage.py migrate
```
#### Загрузить пользователей и роли
```
python manage.py loaddata users.json
```
#### Загрузить посты
```
python manage.py loaddata posts.json
```
#### Запуск сервера
```
python manage.py runserver
```
